# 雅思考点词背诵网站 · 阅读 538 + 听力 179

一个**单文件、免安装、可离线**的雅思高频考点词背诵网页。手机、电脑、平板打开就能用，专为英语基础薄弱、想**集中突破雅思核心词**的同学设计。

内置两个词库，可随时切换：

- **📕 阅读 538**：阅读高频考点词 + 真题同义替换词（约 538 个）
- **📗 听力 179**：听力高频考点词 + 真题同义替换说法（179 个）

背词周期**完全由你决定**——3 天冲刺、7 天稳扎、还是 14 天慢记，选好天数后程序自动把词平均分到每一天。两个词库的进度**各自独立、长期保存**。

> 词表来源：刘洪波《剑桥雅思阅读考点词真经》《剑桥雅思听力考点词真经》。本项目仅是供个人学习使用的**背词工具**，词义与同义替换的版权归原作者与出版方所有，强烈建议购买正版图书系统学习。

## ✨ 它能做什么

- **🔀 双词库切换**：首页顶部一键在「阅读 538」「听力 179」之间切换，各自显示掌握进度，互不干扰。
- **🗓 自选背词周期**：不再固定 5 天。进入词库或点首页「调整」即可选 3 / 5 / 7 / 10 / 14 天或自定义天数，程序按你的天数重新排计划，**改周期不会丢已背记录**。
- **🃏 考点卡片**：每个考点词配中文释义、音标（听力词也含音标）、记忆小窍门。翻到背面会展开**真题同义替换词小卡**——这正是雅思最爱"换着考"的词，每张小卡都能**点开有道查释义和用法**。
- **🔊 真人发音**：点喇叭即可发音（接有道真人发音，可在设置里切换英音 / 美音），比浏览器机器音更稳更准。
- **🎮 闯关游戏**：配对消除、选择题（含"同义替换"专项模式）、单词拼写，边玩边记，两个词库都能玩。
- **📖 英文小故事**：5 篇原创小故事把阅读考点词自然串进去，故事里的目标词都能**点一下查词**，在语境里记得更牢。（小故事目前为阅读词库专属）
- **🗂️ 三色词集**：每次背词选「🔴 不认识 / 🟡 模糊 / 🟢 记得」，系统会**自动把结果汇总成三个词集**，考前一天可以只攻红色和黄色，复习极其高效。
- **⭐ 收藏夹 & 🩹 错题本**：重点词一键收藏；游戏 / 测试做错的词自动进错题本。
- **💾 进度长期保存**：两个词库的背词进度、收藏、三色词集、错题**各自独立记住**，关掉再打开依然在；还能**导出备份文件**换手机时一键导入。

## 🚀 怎么把它发布成网站给别人用（GitHub Pages · 零基础版）

GitHub Pages 是 GitHub 提供的**免费**网页托管，发布后会得到一个 `https://...` 网址，谁都能打开。**全程不用写代码**，跟着做大约 10 分钟。

### 第 1 步：注册 GitHub 账号
打开 https://github.com ，点右上角 **Sign up** 注册（已有账号就登录）。

### 第 2 步：新建一个仓库（repository）
1. 登录后点右上角 **「+」→ New repository**。
2. **Repository name** 随便填，比如 `ielts538`。
3. 选 **Public**（公开，这样别人才能访问）。
4. 点最下面绿色按钮 **Create repository**。

### 第 3 步：上传 index.html
1. 在刚建好的仓库页面，点 **「Add file」→ Upload files**。
2. 把本项目里的 **`index.html`** 拖进去（README、LICENSE 也可以一起拖，不影响）。
3. 等文件上传完，点下面绿色按钮 **Commit changes**。

### 第 4 步：打开 Pages 开关
1. 进入仓库的 **Settings**（设置）。
2. 左侧菜单找到 **Pages**。
3. **Source** 选 **Deploy from a branch**；**Branch** 选 **main**（或 `master`），文件夹保持 **/(root)**，点 **Save**。

### 第 5 步：拿到你的网址
等 1–2 分钟刷新 Pages 页面，顶部会出现一条网址，形如：

```
https://你的用户名.github.io/ielts538/
```

**这个网址就是你的背词网站**，手机电脑都能开，发给同学即可。把它**加到手机主屏幕**（Safari/Chrome 分享 → 添加到主屏幕），用起来就像一个 App。

> 💡 之后想改内容，只要在仓库里重新上传新的 `index.html` 覆盖旧的，网站一两分钟后自动更新。

## 💾 关于"进度保存"（重要）

进度保存用的是浏览器自带的本地存储（localStorage），数据存在**你自己的设备上**，不上传任何服务器，安全私密。**「阅读 538」和「听力 179」两个词库的进度各自独立记录、长期保存**，切换词库或调整周期都不会丢失。

需要注意：

- ✅ **用上面 GitHub Pages 的网址打开时，进度会正常保存**（这是推荐方式）。
- ⚠️ 如果是**直接双击本地的 `index.html` 文件**打开（地址栏是 `file://`），部分手机浏览器会限制本地存储，可能保存不稳定 —— 所以请优先用发布后的网址。
- 📌 换设备、清理浏览器、或换浏览器后数据不互通。可在 **我的 → 进度备份与设置 → 导出**，存一个备份文件（**包含两个词库的全部进度**），到新设备**导入**即可恢复。
